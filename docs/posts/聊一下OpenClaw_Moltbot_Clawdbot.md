## 1\. 解决的问题是什么

Clawdbot最本质要解决的那个问题依旧是——如何让更多人以更低的门槛、更便捷的方式，快速创建自己的专属Agent/工具，从而解决自己的定制化/个性化问题。

在这个大前提下——拖拽式workflow（扣子、dify）、vibe coding、Cursor、Claude Agent SDK/Cowork，解决的都是该问题。只不过在具体的解决方案上，各家的路径、形态、定位各有侧重，但殊途同归，演进的方向是一致的——**人机协作的门槛越来越低，完成的效果/能力越来越强大。**

**核心目标是让“AI”能干活，并让普通人用的上。**

**现在来看，ClawdBot这个Agent平台具备的能力已经是：**

1\. 上手成本越来越低：近2～3年，交互方式的演变路径是从敲代码 → 拖拽流程 → 自然语言描述 → 发消息聊天/后台自动执行所得。这里补充一句对于workflow（此处特指托拉拽节点编排的产品形态）的看法，出发点是解决大模型在生产力的确定性落地上的“指令不遵循”问题，中间包含了人为的规划拆解、工具调用和逻辑执行等。这不是Agentic的做法，是上一代的产物，因为这类产品形态虽然不用写代码，但是上手成本依旧过高（我自己就很难搞懂那些复杂的节点编排）。所以从这类上来说，既然自然语言现在已经能很好地传达，那么用对话直接生成工具，才更符合直觉。2\. 核心能力层：从早期的纯文本生成，到多模态；再到现在环境的感知和动态的规划与执行。从只聊天，简单工具调用到系统级的自主权执行——能看浏览器、能跑 Shell、能定时、能多智能体协作。3\. 覆盖人群越来越广：使用用户的群体扩大了，从开发者到技术爱好者再到普通用户，上手的成本也逐渐降低，从弟订阅+编程，到拖拽&自然语言编程，终于到了自托管时刻。4\. 从“说”到“做”，交付结果越来越强大：从Answer- APP- Solution的转变，从纯文本的给出一个答案给出一个代码片段，再到给出一个可交互的网页，再到直接解决问题。5\. 渠道和入口：虽然是非常常规甚至于说在之前的Agent平台就已经是常规功能的多渠道集成——但我还是想感慨一句，离更强大的Agent时刻终于还是到了，过往的操作思路基本是需要使用特定的客户端（如Cursor）、独立的网页——但是只有把 Agent 嵌入到用户最高频使用的聊天窗口里，这种“触手可及”的便捷性，才是产品爆发的关键。我始终记得一句话：商业的本质是让更多的人以更低廉的成本、更便捷的方式获得更优质的服务。2\. 此类解决方案的演变趋势近3年来，演进趋势主线有以下几个维度：**1\. 核心驱动力** ：让**更多人** （从开发者 → 技术爱好者 → 普通用户）以**更低成本** （从订阅+编程 → 拖拽/自然语言 → 一键自托管）**快速创建** 强大Agent/工具。2\. 人-AI协同演变：人主导 → 人+AI共同构建（拖拽/自然语言编程） → AI主导执行、人轻量意图输入（聊天监督）。3\. 交互便捷性：从敲代码 → 拖拽流程 → 自然语言描述 → 发消息聊天/后台自动。4\. 实现能力越来越多：从只聊天 → 简单工具调用 → 代码/文件自动化 → 系统级自主执行（浏览器、shell、定时、多代理）。大致可分为以下时间节点：

  * 2023：工具调用 + GPTs → 首次“执行”能力。

  * 2024：拖拽workflow + vibe coding → 非开发者AI协同创建工具。

  * 2025：Cursor/Claude Agent → 实时人-AI协同编码/任务。  

  * 2026：Moltbot/Claude Cowork → 普通人拥有“数字员工”，交互能力覆盖生活全场景。

  
维度| 阶段1: 纯对话时代 (2022-2023)| 阶段2: 工具+流程时代 (2023-2024)| 阶段3: 编码/协同辅助时代 (2024-2025)| 阶段4: 自主代理时代 (2025-2026+)  
---|---|---|---|---  
**解决思路**|  被动聊天，文字问答| Chat + 工具调用 + 流程编排| 自然语言/协同驱动代码/任务生成| 完整Agent循环 + 人轻量监督  
**典型产品形态**|  ChatGPT、Claude聊天界面| GPTs、LangChain、dify、n8n+AI| Cursor、Windsurf、vibe coding、Claude Agent SDK| Claude Cowork、Moltbot、SuperAGI、OpenDevin  
**核心能力演变**|  文字/多模态输出，单轮对话| 多模态 + 简单执行（API/浏览器）| 代码生成/实时编辑 + 文件操作| 持久记忆、长时规划、多代理协作、后台主动执行（**能力大幅扩展** ）  
**人/AI协同与门槛降低**|  人完全主导（输入提示），门槛低但能力弱| 人设计流程/工具链（拖拽或简单代码），中等门槛，更多人可参与| 人用自然语言描述意图，AI生成/编辑代码/任务（vibe coding、Cursor协同），门槛大幅降低，非开发者可快速创建工具| 人仅聊天/消息监督，AI自主执行复杂工作流（一键安装+自然语言配置），门槛极低，普通人几分钟即可拥有个人Agent（**更多人、更低成本、快速创建** ）  
**交互与主动性**|  同步对话，用户全驱动| 半同步，用户设定后半自动| 实时协同交互（光标级反馈）| 异步消息驱动 + 后台主动（提醒、定时）（**交互最便捷** ）  
**隐私与控制权**|  云端，完全依赖厂商| 部分本地工具，流程可自控| 本地编辑器，代码/文件完全掌控| 本地自托管/沙盒，数据不离设备

**5\. 聊下其他的（关于未来的观点）**

1\. 所谓平台可能不再是单一工具，而是**操作系统， 跨设备无缝运行（手机、电脑、AR眼镜、甚至机器人），起码在2025年底，这类事情已经见端倪和布局。**2\. 这类平台/OS可能具备什么能力：1）更深度的多模态和物理环境感知、执行融合；2）自我改进与更内化的记忆；3）coding能力，无论如何，coding Agent都是不可或缺的，且由于coding获得了人类的高质量语料库，发展能力会越来越强。3\. 需求有两种：一种是for 个人的个性化的日抛的Demo需求；一种是生产级。前者会卷不胜卷，卷不胜卷，卷不胜卷。但生产级依旧有较大的壁垒。数据、数据、数据；业务知识、业务知识、业务知识——当然现在有个更时髦的说法，skills。4\. 随着超级个体越来越多、普通人搓工具越来越容易，会出现井喷式的AIGC内容，将迎来它的繁荣时间，也将迎来它的垃圾时刻——如何在垃圾时刻中保持判断和思考、是极为重要的——因为你只有训练自己的思考和判断，你才能和AI协同，而不是被AI完全带走。5\. 珍惜人类时代的田园牧歌～珍惜自然与生命本身。无论如何，世界都不是只有AI，还有更广阔的天地。
