# 和 DeepSeek & MiniMax M1 讨论几个产品问题

*本文和 DeepSeek R1 & MiniMax M1 共同完成。*

---

## 一、本文核心观点

### 1. 模型即产品

各大厂的通用 Agent 其本质是通过高频交互场景获取用户数据，反哺模型迭代并构建生态壁垒；形态类似一年前对话机器人的"套壳"，即需通过快速试错验证用户需求。

![模型即产品](/images/wechat/和DeepSeek___MiniMax_M1讨论几个产品问题_259722b29ad5.jpg)

### 2. 随着模型能力增强，当前做的哪些事情可能被颠覆

下一步模型需突破：
- **多步推理能力**（如 OpenAI o3、DeepSeek R1）
- **工具调用可靠性**（如准确调用 API）
- **上下文记忆**（如长文本处理）

随着多模态与推理能力提升，Agent 架构中的**可替代组件**与**不可替代组件**将加速分化：

| 层级 | 当前实现方式 | 未来演进方向 | 颠覆风险 |
|------|-------------|-------------|---------|
| 任务规划 Planning | 提示工程引导模型拆解任务 | 模型内化规划能力（如 GPT-5） | ⭐⭐⭐⭐ |
| 工具调用 Tools | 预设 API 接口+函数调用 | 模型自主生成工具代码 | ⭐⭐ |
| 记忆管理 Memory | 向量数据库+上下文窗口 | 模型长上下文+参数化记忆 | ⭐⭐⭐ |
| 执行闭环 Action | 人工配置工作流 | 自主感知环境+动态决策 | ⭐ |

### 3. 用户想要的是什么，一些潜在的未来趋势

绝大多数用户对于模型是没有概念的，需求可能是日常的搜索、润色下文案、翻译——所以对于此类用户，哪个好用用哪个、用的习惯用哪个，反正都差不多。

所以，对于产品而言：
- **无感化嵌入生活流**
- **情绪价值 > 功能价值**

简单粗暴的解法是：如果要流量和更广泛的用户，融入当前的超级 APP，降低认知与操作成本，比独立 APP 存活率更高。

重视效果的用户（相对专业），眼下主要作用是提效。但提效不是终点，**效果可量化是刚需**。

### 4. 一些反常识洞见

- 对小白用户，"通用平台"反而更垂直（微信/抖音即他们的全部世界）
- 对专业用户，"垂直工具"反而更通用（GitHub Copilot 覆盖前端/算法/运维多角色）

### 5. AI 不能取代的是什么

再次回归和两位 LLM 交互的过程，问题相对开放，最终也是在两位的基础之上整合而成，他们也都给了我新的灵感和启发。就整体而言，DS 略微更好，但是 MiniMax M1 也不差。

他们说的很好，但我需要鉴别和判断——所以最终获取接近正确的答案应该类似于圆桌会议的 Multi-Agent 交流方式——很类似于人的方式。

但我接着意识到，**AI 不能替代人的是什么——是连接**。

对，是连接——类似问题我也可以和人一起讨论，也许人给出的答案不如 AI 完美，但是人给到的体验却是 AI 给不到的——**是鲜活，是从表情、神态、动作、眼神、声调中所给予的连接**。

> **人与人的关系连接，至少当下，始终不能被 AI 替代。**
> 
> 等能被替代了，爱上机器人这类事情大概也会发生了 🤣

## 二、我的问题

现在每个模型都在做"通用 Agent"，如 code Agent、网页 Agent；其形态类似一年前，每家的模型产品都有多个对话机器人。

**有以下猜测：**

1. 本质是引流获取数据、入口，从而增强下一代模型的手段，现在的情况，模型即产品
2. 这依旧只是一个短暂的阶段，最重要的事情是增强模型的能力——那么在这个阶段中，哪些会被模型能力增强而颠覆？
3. 类比之前机器人的情况，中美用户在哪些方向和场景上有需求、有付费意愿？

---

## 三、我希望得到的是

1. 对于模型即产品的观点作证
2. 当下的哪些 AI 建设能力是模型的补位，即下一代模型能力提高就会将之颠覆
3. 用户想要的是什么，针对潜在的需求场景，可以在哪些点发力

> **有趣的是**，在我提问的时候，我并未太深入想我想要获得的是什么，是在判断两位 AI 谁回答的更好的过程中发现的。
>
> 和 AI 交互的过程，本身就是能帮助验证和判断的过程。而与 AI 交互，最重要的是两类能力：**提问**以及**判断标准**。很多时候，问题是开放性的，判断标准的本质是选择，也就是审美。

## 四、MiniMax M1 和 DeepSeek R1 的答案

### 【MiniMax M1】

#### 1. 引流与数据入口：模型即产品的本质

当前"通用 Agent"的爆发式出现，核心逻辑符合**"模型即产品"**的短期战略，其本质是通过高频交互场景获取用户行为数据，反哺模型迭代并构建生态壁垒：

- **数据驱动模型进化**：大模型能力提升依赖海量高质量数据，而 Agent 作为"数字员工"能直接嵌入用户工作流，积累真实场景数据
- **入口争夺战**：Agent 作为新一代人机交互界面，类似移动互联网时代的"超级 APP"，成为用户接触 AI 的核心入口
- **短期验证阶段**：当前形态类似一年前对话机器人的"套壳"，需通过快速试错验证用户需求

#### 2. 模型能力增强的颠覆方向

AI Agent 的真正价值在于**通过模型能力突破重构传统范式**：

**编程与软件开发**
- 代码生成、调试、版本管理全流程自动化（如 Devin、Cursor）
- 未来可能替代部分初级程序员，聚焦架构设计与复杂逻辑处理

**企业服务与 B 端场景**
- 智能客服：从简单问答升级为复杂流程处理
- 数据分析：自动清洗数据、生成可视化报告
- 文档处理：合同分析、报告生成效率提升 3-5 倍

**个人生产力工具**
- 智能日程管理、邮件处理、会议纪要生成
- 跨平台工具调用，实现端到端自动化

**颠覆前提**：模型需突破**多步推理能力**、**工具调用可靠性**和**上下文记忆**。

#### 3. 中美用户需求与付费意愿对比

| 维度 | 美国市场 | 中国市场 |
|------|---------|---------|
| **优势赛道** | 图像生成、虚拟角色、音乐、开发者工具 | Chatbot、教育、通用 Agent |
| **用户习惯** | PC 端为主，付费意愿强 | 手机端为主，付费习惯待培养 |
| **付费驱动** | 情感陪伴、专业工具提效 | 本土化服务、性价比 |

### 【DeepSeek R1】

#### 一、Agent 的本质：数据入口与模型能力的"双重载体"

**短期战略：数据飞轮与场景卡位**

当前通用 Agent 本质是**模型厂商的生态入口**：通过提供标准化 Agent 框架，吸引开发者构建垂直应用，反哺训练数据与 API 调用量。

**长期瓶颈：脱离场景的通用化是伪命题**

Agent 的核心价值在**任务闭环效率**，而非"万能"标签。**过度通用化将导致体验割裂**：编程 Agent 需深度理解代码上下文，而客服 Agent 需情感推理能力，二者底层模型优化方向迥异。

#### 二、技术演进：模型能力增强将颠覆的中间层

- **高颠覆领域**：任务规划（Planning）依赖的提示工程将被模型原生推理能力取代
- **低颠覆领域**：物理世界交互需专用硬件接口与环境传感器，模型无法纯软件替代

#### 三、中美需求差异

**中国市场的爆发点**
- 企业降本场景：金融、政务领域的流程自动化需求明确
- 硬件协同优势：机器人硬件+Agent 软件捆绑销售
- 教育普惠：AI 辅导 Agent 因教育资源不均获得家长高付费意愿

**美国市场的付费逻辑**
- 人力替代溢价：会议总结、编程助手类工具因节省高成本人力受追捧
- 虚拟消费文化：虚拟偶像和 NSFW 内容驱动 C 端订阅
- 开发者生态：工具链整合平台因接入数千 API 形成生态壁垒

**需求差异核心**
- **中国企业重"结果可验证"**：要求 Agent 输出直接对应降本指标
- **美国用户重"过程自主性"**：倾向为"全托管"服务付费

#### 四、未来演进：Agent 形态的终局判断

1. **入口价值让位于闭环价值**：通用 Agent 的引流作用将随模型开源弱化，真正留存用户的是垂直场景的任务完成度

2. **多 Agent 协作成为主流范式**：单 Agent 框架在复杂任务中易出错，未来 Agent 将按功能模块化

3. **硬件成为关键落地载体**：机器人 AI 临界点突破后，通用 Agent 需与传感器、机械臂深度耦合

---

## 结论

当前通用 Agent 本质是模型厂商的数据战略工具，其形态将在 2 年内被模型原生能力（规划/记忆）颠覆。

长期胜出的 Agent 需在**垂直闭环效率**或**硬件协同**上构建壁垒。

中企应聚焦企业降本与教育普惠场景，避免在虚耗性"通用标签"中内卷。
