# 🍢 No.7 胃之书体验

<small>2024.08.04 · 来自即刻「AI探索站」</small>

---

## 整体感受

1. **体验上比较炫酷**，尤其模型处理的等待过程流式输出 JSON 配合震动，科技感拉满 ✨

2. **功能定位**：以识别热量给出营养学建议为主，像是营养助手。其中烟火圈支持好友（社交属性）、八大菜系解锁（这个功能没看清楚，是促进用户上传菜单从而收集更丰富的餐食数据吗？）

3. **热量估计不准**。在这种情况下必然支持编辑，编辑后热量算的比较准了。所以识别菜品以及其重量是关键点

4. **稳定性一般**。居然出现告诉我重试的情况，且重试后毫无反应。另外识别菜品也出现幻觉，同一张图片两次菜品识别不一样

## 一些想法

### 1. 适用用户

比较适合对营养和热量没太多概念的选手。

> 以自己这种常年计算热量的用户为例，模型还没我估的准。。😅

### 2. 如何会更准

识别菜品和重量是关键。最好的方式当然是后厨直接把重量写进去，但不现实。

- 可以初步估计总重量（给用户参考，比如多长尺寸的盘子多少 g 之类），然后按照比例计算
- 或者是在拍照的时候自动估尺寸

### 3. 如果不准怎么办

识别餐品相对容易和准确，与其给热量不如给营养学建议：

> 比如这顿炸鸡吃多了建议下顿吃点蔬菜 🥬，这段时间吃的较为油腻建议增加训练

相对容易也更符合健康原则 —— **重要的不是热量而是吃什么**。
